# F1.2 - W03 - Frontend Home Feature

> **Feature:** F1.2 - Home Dashboard
> **Release:** 1.0 | **Sprint:** S01
> **Type:** frontend | **Priority:** High
> **Estimate:** 5 story points
> **Assignable to:** Frontend Dev

---

## Description

Build the Angular 19 Home feature (`/inicio`) using AppKit 4 components and Angular Signals. The
page is the authenticated user's landing screen after login; it shows recent activity, monitoring
alerts, a projects placeholder, and quick-access navigation.

---

## Tasks

### Feature scaffold (use `schema-la` schematic)

- [ ] Run `ng generate schema-la:feature home --project features` (or manually create the folder)
    - Creates `frontend/projects/features/home/` with standard structure (components, services, models, routes)

### Models

- [ ] **Create** `frontend/projects/features/home/src/lib/models/home.model.ts`
    ```typescript
    export interface HomeData {
      recentActivity: RecentActivityItem[];
      latestAlerts: AlertSummaryItem[];
      projects: ProjectSummaryItem[];
    }
    export interface RecentActivityItem { kind: string; title: string; entityId: string; accessedAt: string; }
    export interface AlertSummaryItem   { alertId: string; title: string; branch: string; publishedAt: string; }
    export interface ProjectSummaryItem { projectId: string; name: string; lastActivityAt: string; }
    ```

### Service

- [ ] **Create** `frontend/projects/features/home/src/lib/services/home.service.ts`
    - `private readonly _data = signal<HomeData | null>(null)`
    - `readonly data = this._data.asReadonly()`
    - `readonly loading = signal(false)`
    - `load(): void` — `GET /api/home` → `_data.set(response)` — called on route activate

### Components

- [ ] **Create** `home-shell.component.ts`
    - Greeting header: "Buenas tardes, {displayName}" (from `AuthService.identity()`)
    - Bell icon with unread alert count badge
    - Grid layout: recent activity (left) + alerts (right) + projects row + quick-access row
    - Calls `homeService.load()` on `ngOnInit`

- [ ] **Create** `recent-activity.component.ts`
    - Receives `RecentActivityItem[]` as input signal
    - Renders AppKit 4 list component; each item shows icon (by `kind`), title, relative time
    - Clicking an item navigates to `/normas/{id}`, `/jurisprudencia/{id}`, or `/asistente` by kind

- [ ] **Create** `alert-preview.component.ts`
    - Receives `AlertSummaryItem[]` as input signal (up to 5)
    - AppKit 4 compact card per alert; branch chip; date; "Ver todas" link → `/alertas`
    - Empty state: "No hay alertas nuevas"

- [ ] **Create** `project-list-preview.component.ts`
    - Receives `ProjectSummaryItem[]` as input signal
    - Shows up to 4 project cards in a grid row; each card: name + last activity date
    - Empty state: "Los proyectos estarán disponibles próximamente" (R2.0 placeholder)
    - "Ver proyectos" link disabled (tooltip: "Disponible en R2.0")

- [ ] **Create** `quick-access.component.ts`
    - Three AppKit 4 primary buttons:
      - "Buscar normas" → `/normas`
      - "Buscar jurisprudencia" → `/jurisprudencia`
      - "Asistente IA" → `/asistente`

### Routing

- [ ] **Create** `frontend/projects/features/home/src/lib/home.routes.ts`
