## 1. Feature Overview

The Home screen is the **per-user landing page** after login. It replaces the generic MVP dashboard
with a product-relevant view oriented around the user's daily PwC tax-legal work.

**MVP delta:** The MVP has a generic dashboard. This feature re-scopes it to show: recently accessed
items, pending tasks, regulatory alerts, and quick-access to the main KB search areas.

> Note: "My Projects" (workspace card grid) is introduced in **F2.1 - Projects/Workspaces** (R2.0).
> The Home feature in R1.0 shows a simplified project list (name + last activity) as a placeholder,
> with the full project UX coming in R2.0.

## 3. Backend Architecture

### 3.1 Endpoint

| Method | Route          | Auth        | Description               |
| ------ | -------------- | ----------- | ------------------------- |
| GET    | `/api/home`    | PwCStaff    | Returns full home payload |

```csharp
// Contracts/Home/HomeResponse.cs
public sealed record HomeResponse(
    IReadOnlyList<RecentActivityItem> RecentActivity,
    IReadOnlyList<AlertSummaryItem>   LatestAlerts,
    IReadOnlyList<ProjectSummaryItem> Projects);      // placeholder — full data in F2.1

public sealed record RecentActivityItem(
    string Kind,        // "search" | "norm" | "ruling" | "chat"
    string Title,
    string EntityId,
    DateTimeOffset AccessedAt);

public sealed record AlertSummaryItem(
    Guid   AlertId,
    string Title,
    string Branch,      // tax branch / law branch
    DateTimeOffset PublishedAt);

public sealed record ProjectSummaryItem(
    Guid   ProjectId,
    string Name,
    DateTimeOffset LastActivityAt);
```

### 3.2 Data sources

- **Recent activity:** `UserActivity` table (see §4) — last 10 records for the current user.
- **Alerts:** first 5 unread alerts from `MonitoringAlerts` (introduced in F1.8). If F1.8 is not yet
  deployed, return empty list.
- **Projects:** simple `SELECT Id, Name, LastActivityAt FROM Projects WHERE userId = @userId` —
  populated when F2.1 lands; returns empty list until then.

### 3.3 UserActivity entity (new in this feature)

```sql
CREATE TABLE UserActivity (
    Id            BIGINT         PRIMARY KEY IDENTITY,
    UserId        INT            NOT NULL REFERENCES UserPreferences(Id),
    Kind          NVARCHAR(20)   NOT NULL,   -- search|norm|ruling|chat
    Title         NVARCHAR(500)  NOT NULL,
    EntityId      NVARCHAR(100)  NULL,       -- norm/ruling id
    AccessedAt    DATETIME2      NOT NULL DEFAULT GETUTCDATE()
);
CREATE INDEX IX_UserActivity_UserId_AccessedAt ON UserActivity (UserId, AccessedAt DESC);
```

Activity rows are written by the search and detail endpoints (fire-and-forget via domain event or
direct insert — team decides at implementation time).

## 4. Frontend Architecture

```

├── components/
│   ├── home-shell/              # Page wrapper + greeting
```
