# F1.8 - W03 - Frontend Monitoring Settings and Alerts

> **Feature:** F1.8 - Regulatory and Tax Monitoring & Alerts
> **Release:** 1.0 | **Sprint:** S01
> **Type:** frontend | **Priority:** Medium
> **Estimate:** 5 story points
> **Assignable to:** Frontend Dev

---

## Description

Build the Angular 19 monitoring feature at `/alertas`. Subscription management form + alert inbox
with read/unread state. SignalR integration for real-time badge updates in the app header.

---

## Tasks

### Feature scaffold

- [ ] Generate `frontend/projects/features/monitoring/` via `schema-la:feature`

### Models

- [ ] **Create** `monitoring.model.ts`
    ```typescript
    export interface CreateSubscriptionRequest {
      name: string; lawBranch?: string; taxBranch?: string;
      issuingBody?: string; keywords?: string; sourceTypes: string[];
    }
    export interface SubscriptionResponse { id: string; name: string; isActive: boolean; /* ... */ }
    export interface AlertResponse {
      alertId: string; itemType: string; itemId: string; title: string;
      matchedBranch?: string; isRead: boolean; publishedAt: string; createdAt: string;
    }
    ```

### Services

- [ ] **Create** `monitoring.service.ts`
    - `readonly subscriptions = signal<SubscriptionResponse[]>([])`
    - `readonly alerts = signal<AlertResponse[]>([])`
    - `readonly unreadCount = signal(0)`
    - `readonly loading = signal(false)`
    - `loadSubscriptions()`, `createSubscription(req)`, `updateSubscription(id, req)`, `deleteSubscription(id)`
    - `loadAlerts(page?)`, `markRead(alertId)`, `markAllRead()`
    - `loadUnreadCount()` — called on app init and after SignalR push

- [ ] **Create** `signalr-alerts.service.ts`
    - Connects to SignalR hub (URL from environment)
    - On `NewAlert` event: calls `monitoringService.loadUnreadCount()`; optionally push toast via AppKit 4
    - Reconnect on disconnect (exponential backoff)
    - Started in `APP_INITIALIZER` after auth bootstrap

### Components

- [ ] **Create** `monitoring-shell.component.ts`
    - Two-tab layout (AppKit 4 tabs): Alertas | Mis Suscripciones
    - "Alertas" tab shows `alert-list`; "Mis Suscripciones" shows `subscription-list`

- [ ] **Create** `alert-list.component.ts`
    - Paged list of `alert-item` components
    - "Marcar todas como leídas" button (disabled if no unread)
    - Empty state: "No tienes alertas. Configura suscripciones para recibirlas."
    - Unread alerts shown with bold title + blue left border

- [ ] **Create** `alert-item.component.ts`
    - Item type icon (norm/ruling/tax-source)
    - Title (bold if unread); matched branch chip; date
    - Click: mark as read + navigate to detail page based on `itemType`
    - Swipe-to-dismiss (optional nice-to-have)

- [ ] **Create** `subscription-list.component.ts`
    - Table of subscriptions: name, filters summary, active toggle
    - "Nueva suscripción" button → opens `subscription-form` dialog
    - Edit and delete icons per row

- [ ] **Create** `subscription-form.component.ts` (dialog/panel)
    - Fields: Name (required), Law branch (select from known branches), Tax branch (select),
      Issuing body (text), Keywords (tag input), Source types (multi-checkbox: normas/jurisprudencia/fuentes tributarias)
    - Validate: at least one filter field required
    - Save / Cancel buttons

- [ ] **Create** `alert-badge.component.ts` (shared component in `core/` or `shared-common/`)
    - Binds to `monitoringService.unreadCount` signal
    - Shows red dot with count (AppKit 4 badge pattern)
    - Placed in the AppKit 4 shell header (bell icon)
