### Infrastructure — Alert evaluation worker

- [ ] **Create** `backend/src/shared/LegalAiAr.Infrastructure/Workers/AlertEvaluationWorker.cs`
    - `BackgroundService`; registered in API process
    - Timer: configurable interval (default 30 min) via `appsettings: Monitoring:EvaluationIntervalMinutes`
    - **Checkpoint:** reads `SystemSettings["AlertWorker:LastRunAt"]`; updates after each run
    - **Query new items:** `SELECT * FROM TaxDocuments WHERE IngestedAt > @lastRun`
      + `SELECT * FROM LegalNorms WHERE IngestedAt > @lastRun` + `SELECT * FROM CaseLaw WHERE IngestedAt > @lastRun`
    - **Match logic:**
      ```
      For each new item × each active subscription:
        if (sub.Organism != null && item.Organism != sub.Organism) → skip
        if (sub.LawBranch != null && item.LawBranch != sub.LawBranch) → skip
        if (sub.TaxBranch != null && item.TaxBranch != sub.TaxBranch) → skip
        if (sub.Keywords != null && !item.Title.ContainsAny(keywords)) → skip
        if (sub.SourceTypes not contains item.ItemType) → skip
        → insert MonitoringAlert
      ```
    - **Deduplication:** skip if `MonitoringAlert` with `(SubscriptionId, ItemId)` already exists
    - **SignalR push** after inserting: `_hubContext.Clients.Group("userId-{userId}").SendAsync("NewAlert", …)`
