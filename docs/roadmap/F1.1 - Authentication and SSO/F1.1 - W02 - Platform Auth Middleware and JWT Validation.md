# F1.1 - W02 - Platform Auth Middleware and JWT Validation

> **Feature:** F1.1 - Authentication & Corporate SSO
> **Release:** 1.0 | **Sprint:** S01
> **Type:** backend | **Priority:** Critical (blocking all R1 features)
> **Estimate:** 5 story points
> **Assignable to:** Backend Dev

---

## Description

Replace the dev/simulated auth stub with the **production platform-managed Entra `id_token` cookie
validation** as specified in
[PwC Internal Application Architecture §1 and §5](../../standards/pwc-internal-app-architecture.md).

The backend must validate the JWT from the HTTP-only cookie, extract user claims (OID, email,
display name, roles), and surface them via `GET /api/auth/me`. All other protected endpoints must
return 401 when the cookie is absent or invalid.

## Tasks

### Infrastructure — Auth handler

- [ ] **Create** `backend/src/shared/LegalAiAr.Infrastructure/Auth/PlatformAuthOptions.cs`
    - Properties: `Issuer`, `Audience`, `CookieName` (default `"id_token"`), `ValidateLifetime` (bool)
- [ ] **Create** `backend/src/shared/LegalAiAr.Infrastructure/Auth/PlatformAuthHandler.cs`
    - Inherits `AuthenticationHandler<PlatformAuthOptions>`
    - Reads cookie `context.Request.Cookies[Options.CookieName]`
    - Validates JWT with `TokenValidationParameters` (issuer, audience, signature via OIDC discovery)
    - Sets `ClaimsPrincipal` on success; returns `AuthenticateResult.Fail` on missing/invalid token
    - Claim mapping: `oid` → user ID, `email`/`preferred_username`, `name`, `roles[]`
- [ ] **Create** `backend/src/shared/LegalAiAr.Infrastructure/Auth/AuthServiceExtensions.cs`
    - `AddPlatformAuth(this IServiceCollection, IConfiguration)` — registers handler + options
    - Registers `AuthorizationPolicies`: `AdminOnly` (role `admin`), `PwCStaff` (roles partner/senior/associate/admin)
- [ ] **Modify** `backend/src/api/LegalAiAr.Api/Program.cs`
    - Replace `AddAuthentication().AddJwtBearer(...)` stub with `AddPlatformAuth(configuration)`
    - Ensure `UseAuthentication()` + `UseAuthorization()` are called in correct order

### Contracts — UserIdentityResponse

- [ ] **Create** `backend/src/shared/LegalAiAr.Contracts/Auth/UserIdentityResponse.cs`

        string Id,

        string DisplayName,

### Configuration
