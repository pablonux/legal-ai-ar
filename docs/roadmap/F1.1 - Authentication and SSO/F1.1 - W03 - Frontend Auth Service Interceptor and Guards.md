# F1.1 - W03 - Frontend Auth Service, Interceptor and Guards

> **Feature:** F1.1 - Authentication & Corporate SSO
> **Release:** 1.0 | **Sprint:** S01
> **Type:** frontend | **Priority:** Critical (blocking all R1 frontend work)
> **Estimate:** 5 story points
> **Assignable to:** Frontend Dev

---

## Description

Wire up the Angular 19 SPA to the platform auth model: bootstrap the user identity via
`GET /api/auth/me`, handle 401 redirects, protect routes with guards, and expose role-based
directives. No MSAL, no Bearer tokens. All HTTP calls use `withCredentials: true`.

**Prerequisite:** F1.1-W02 must be deployed to DEV so `/api/auth/me` returns real data.

## Tasks

### Auth service (`core/auth/`)

- [ ] **Create** `frontend/projects/core/src/lib/auth/auth.model.ts`
    ```typescript
    export interface UserIdentity {
      id: string;
      email: string;
      displayName: string;
      role: string;
      roles: string[];
    }
    ```
- [ ] **Create** `frontend/projects/core/src/lib/auth/auth.service.ts`
    - `private readonly _identity = signal<UserIdentity | null>(null)`
    - `readonly identity = this._identity.asReadonly()`
    - `readonly isAuthenticated = computed(() => this._identity() !== null)`
    - `readonly role = computed(() => this._identity()?.role ?? '')`
    - `bootstrap(): Observable<void>` — `GET /api/auth/me` → populate `_identity`; on 401 call `redirectToLogin()`
    - `logout(): void` — `POST /api/auth/logout` → redirect to `logoutUrl` from response
    - `redirectToLogin(): void` — navigate to `environment.ssoLoginUrl`
    - `hasRole(...roles: string[]): boolean` — checks `_identity().roles`
- [ ] **Modify** `frontend/projects/app/src/app/app.config.ts`
    - Add `APP_INITIALIZER` that calls `authService.bootstrap()` before any route is activated

### HTTP interceptor

- [ ] **Create** `frontend/projects/core/src/lib/interceptors/credentials.interceptor.ts`
    - `HttpInterceptorFn` — adds `withCredentials: true` to every outgoing request
    - On `HttpErrorResponse` status 401: calls `authService.redirectToLogin()` (skip for `/api/auth/me` to avoid loops)
- [ ] **Modify** `frontend/projects/app/src/app/app.config.ts`
    - Register `credentialsInterceptor` via `provideHttpClient(withInterceptors([credentialsInterceptor]))`

### Auth guard

- [ ] **Create** `frontend/projects/core/src/lib/guards/auth.guard.ts`
    - `CanActivateFn` — if `!authService.isAuthenticated()` → `authService.redirectToLogin()` + return `false`
    - Applied on all feature routes
- [ ] **Create** `frontend/projects/core/src/lib/guards/role.guard.ts`
    - `CanActivateFn` factory — `roleGuard('admin')` → if authenticated but wrong role → redirect to `/acceso-denegado`

### Role directive

- [ ] **Create** `frontend/projects/core/src/lib/directives/has-role.directive.ts`
    - `@Directive({ selector: '[appHasRole]' })` structural directive
    - Input: `appHasRole: string | string[]`
    - Shows/hides host element based on `authService.hasRole(...)`
    - Example usage: `<button *appHasRole="'admin'">Admin panel</button>`

### Environment configuration

- [ ] **Modify** `frontend/projects/core/src/lib/environments/` — confirm `ssoLoginUrl` exists in all env files:
