## 1. Feature Overview

Production-ready jurisprudence search over CSJN and SAIJ rulings, backed by hybrid search on
`idx-case-law`. MVP coverage is **~90%** — this feature aligns the backend to Contracts/Minimal API
and delivers the polished Angular 19 search UI. The existing `RulingsController` was already migrated
to Minimal API in F0.0-W09 (`RulingsEndpointGroup`).

---

## 2. Search Capabilities

| Capability           | Implementation                                       |
| -------------------- | ---------------------------------------------------- |
| Full-text + semantic | Hybrid search on `idx-case-law` (BM25 + 3072-d vectors)|
| Court filter         | `court` facet + free text                            |
| Court venue filter   | `courtVenue` (civil, laboral, comercial, contadmin…) |
| Instance filter      | `instance` (primera, segunda, csjn, casacion)        |
| Date range           | `rulingDate` range filter                            |
| En banc filter       | `isEnBanc` boolean                                   |
| Topic keywords       | `topicKeywords` multi-value filter                   |
| Sort                 | Relevance (default) / date desc / date asc           |
| Pagination           | Cursor-based, 20 per page                            |

## 3. Backend Architecture

### 3.1 Endpoints (already scaffolded in F0.0-W09)

| Method | Route                          | Auth     | Description                                   |
| ------ | ------------------------------ | -------- | --------------------------------------------- |
| POST   | `/api/rulings/search`          | PwCStaff | Hybrid search over `idx-case-law`             |
| GET    | `/api/rulings/{id}`            | PwCStaff | Ruling detail (→ F1.6 extends)                |
| GET    | `/api/rulings/{id}/document`   | PwCStaff | Stream original PDF from Blob Storage         |
| GET    | `/api/rulings/{id}/related`    | PwCStaff | Related rulings (graph traversal `CitesCaseLaw`)|
| GET    | `/api/rulings/facets`          | PwCStaff | Available facet values with counts            |

### 3.2 Contracts

```csharp
// Contracts/Rulings/SearchRulingsRequest.cs
public sealed record SearchRulingsRequest(
    string?  Query,
    string?  Court,
    string?  CourtVenue,
    string?  Instance,
    bool?    IsEnBanc,
    IReadOnlyList<string>? TopicKeywords,
    DateOnly? RulingDateFrom,
    DateOnly? RulingDateTo,
    string   SortBy = "relevance",
    int      Page = 1,
    int      PageSize = 20);

// Contracts/Rulings/RulingSummary.cs
public sealed record RulingSummary(
    Guid     Id,
    string   Caption,
    string   RulingNumber,
    DateOnly RulingDate,
    string   Court,
    string   CourtVenue,
    string   Instance,
    bool     IsEnBanc,
    IReadOnlyList<string> TopicKeywords,
    string   HighlightedHeadnote);

// Contracts/Rulings/SearchRulingsResponse.cs
public sealed record SearchRulingsResponse(
    IReadOnlyList<RulingSummary> Results,
    int TotalCount,
    IReadOnlyDictionary<string, IReadOnlyList<FacetValue>> Facets,
    string? NextCursor);
```

### 4.2 State

```typescript
@Injectable({ providedIn: 'root' })
export class CaseLawSearchService {
  readonly filters  = signal<SearchRulingsRequest>(defaultFilters);
  readonly results  = signal<SearchRulingsResponse | null>(null);
  readonly loading  = signal(false);

  search(overrides?: Partial<SearchRulingsRequest>): void;
  loadMore(): void;
  clearFilters(): void;
}
```

### 4.3 UI notes

- Topic keyword chips are clickable to add to the active filter.
- Court autocomplete uses `GET /api/courts` (existing endpoint) for suggestions.
- En banc toggle is a prominent switch for CSJN plenario searches.
