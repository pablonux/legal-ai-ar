# F1.3 - W03 - Frontend Case-Law Search Feature

> **Feature:** F1.3 - Case-Law (Jurisprudence) Search
> **Release:** 1.0 | **Sprint:** S01
> **Type:** frontend | **Priority:** High
> **Estimate:** 5 story points
> **Assignable to:** Frontend Dev

---

## Description

Build the Angular 19 case-law search feature at `/jurisprudencia`. Search bar + filter sidebar
(court, venue, instance, en banc, dates, topic keywords) + result cards. AppKit 4 components,
Signals state.

---

## Tasks

### Feature scaffold

- [ ] Generate `frontend/projects/features/case-law-search/` via `schema-la:feature`

### Models

- [ ] **Create** `ruling.model.ts`
    ```typescript
    export interface SearchRulingsRequest {
      query?: string; court?: string; courtVenue?: string; instance?: string;
      isEnBanc?: boolean; topicKeywords?: string[]; rulingDateFrom?: string;
      rulingDateTo?: string; sortBy: string; page: number; pageSize: number;
    }
    export interface RulingSummary {
      id: string; caption: string; rulingNumber: string; rulingDate: string;
      court: string; courtVenue: string; instance: string; isEnBanc: boolean;
      topicKeywords: string[]; highlightedHeadnote: string;
    }
    ```

### Service

- [ ] **Create** `case-law-search.service.ts`
    - Signals: `filters`, `results`, `loading`
    - Methods: `search(overrides?)`, `loadMore()`, `clearFilters()`
    - Autocomplete helper: `suggestCourts(term: string): Observable<string[]>` (calls `GET /api/courts`)

### Components

- [ ] **Create** `case-law-search-shell.component.ts` — two-panel; title "Buscar Jurisprudencia"

- [ ] **Create** `case-law-filters.component.ts`
    - Court autocomplete (AppKit 4 typeahead backed by `/api/courts`)
    - Court venue: checkboxes (civil, laboral, comercial, contadmin, familia, penal)
    - Instance: radio buttons (Primera instancia / Cámara / CSJN / Casación)
    - En banc: toggle switch
    - Date range pickers
    - Topic keywords: tag input — user types terms; each term added as a chip filter

- [ ] **Create** `ruling-result-card.component.ts`
    - Caption (truncated to 2 lines), ruling number + date
    - Court badge, instance chip, en banc star icon
    - Topic keyword chips (up to 4; "+N more" if excess)
    - Highlighted headnote
    - Click → navigate to `/jurisprudencia/{id}`

- [ ] **Create** `case-law-results.component.ts`
    - List of `ruling-result-card`; total count header; pagination; empty state

- [ ] **Create** `ruling-topic-chips.component.ts` (reusable)
    - Receives `string[]`; each chip is clickable to add to active keyword filter

### Routing

- [ ] Route `/jurisprudencia` → search shell (protected by `authGuard`)
- [ ] Route `/jurisprudencia/:id` → (navigates to norm-detail feature — F1.6)

### Tests

- [ ] Unit: topic keyword chip click adds keyword to `filters.topicKeywords`
- [ ] Unit: result card renders en-banc icon for `isEnBanc: true`
- [ ] E2E smoke: search for "CSJN" → results with court badge "CSJN"

### Verification

- [ ] All filter types functional; results update reactively
