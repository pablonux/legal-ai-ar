# F1.6 - W02 - Norm Detail Endpoints and Graph Traversal

> **Feature:** F1.6 - Norm and Article Detail
> **Release:** 1.0 | **Sprint:** S01
> **Type:** backend | **Priority:** Medium-High
> **Estimate:** 5 story points
> **Assignable to:** Backend Dev

---

## Description

Complete the norm and ruling detail endpoints (article browsing, graph traversal) and wire them to
Minimal API + Contracts. Implement `LegalGraphService` for SQL Graph traversal returning a
`GraphResponse` suitable for Cytoscape.js.

### Infrastructure — LegalGraphService

- [ ] **Create** `backend/src/shared/LegalAiAr.Core/Interfaces/ILegalGraphService.cs`
    - `Task<GraphResponse> GetNormRelationsAsync(int normId, int maxDepth = 2, int maxNodes = 50)`
    - `Task<GraphResponse> GetRulingRelationsAsync(Guid rulingId, int maxDepth = 1, int maxNodes = 30)`

- [ ] **Create** `backend/src/shared/LegalAiAr.Infrastructure/Graph/LegalGraphService.cs`
    - Executes SQL Graph `MATCH` queries via `DbContext.Database.SqlQueryRaw<T>` or Dapper
    - Norm query: 2-hop traversal over `Amends`, `Repeals`, `Regulates`, `Interprets`, `Applies`
    - Ruling query: 1-hop over `CitesCaseLaw`, `Applies`, `Interprets`
    - Result truncated at `maxNodes`; if > `maxNodes`, add `"truncated": true` flag to response
    - Maps raw SQL results to `GraphNode` + `GraphEdge` records
