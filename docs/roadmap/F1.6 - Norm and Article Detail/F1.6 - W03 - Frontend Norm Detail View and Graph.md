# F1.6 - W03 - Frontend Norm Detail View and Relationship Graph

> **Feature:** F1.6 - Norm and Article Detail
> **Release:** 1.0 | **Sprint:** S01
> **Type:** frontend | **Priority:** Medium-High
> **Estimate:** 5 story points
> **Assignable to:** Frontend Dev

---

## Description

Build the norm/article detail view with tabs (summary, articles, relationship graph, related
case-law) and the Cytoscape.js relationship graph. Works for both statutes (`/normas/:id`) and
rulings (`/jurisprudencia/:id`).

---

## Tasks

### Feature scaffold

- [ ] Generate `frontend/projects/features/norm-detail/` via `schema-la:feature`

### Service

- [ ] **Create** `norm-detail.service.ts`
    - `getStatuteDetail(id: number): Observable<StatuteDetail>`
    - `getArticles(id: number, page: number): Observable<ArticlePage>`
    - `getRelations(id: number): Observable<GraphData>`
    - `getCaseLaw(id: number): Observable<RulingsPage>`

- [ ] **Create** `ruling-detail.service.ts` (or unify via parametrized approach)

### Components

- [ ] **Create** `norm-detail-shell.component.ts`
    - Route params → load detail on init
    - Tabs (AppKit 4 tabs): Resumen | Artículos | Relaciones | Jurisprudencia
    - Metadata header: norm number, name, type badge, in-force status badge
    - "Consultar con asistente" button → `/asistente?context=norm:{id}`

- [ ] **Create** `norm-metadata-panel.component.ts`
    - Key-value grid: NormType, HierarchyLevel, LawBranch, TerritorialScope, IssuingBody,
      PublicationDate, EffectiveDate, OriginalTextUrl
    - Full-text summary (collapsible if > 500 chars)

- [ ] **Create** `article-browser.component.ts`
    - Virtual scroll (CDK `ScrollingModule`) over articles list — handles norms with 500+ articles
    - In-article search: filter input debounced 300 ms; client-side filter on loaded articles
      (or triggers new query with `query` param if backend supports it)
    - Each item: `article-item.component` with article number chip + normative text

- [ ] **Create** `article-item.component.ts`
    - Article number chip, chapter/section title (if present), full normative text
    - Copy-to-clipboard action

- [ ] **Create** `relation-graph.component.ts`
    - Wraps Cytoscape.js (already in MVP dependencies; confirm `cytoscape` in `package.json`)
    - On `graphData` input change: destroy + recreate graph with `cose` layout
    - Node click → `router.navigate` to `/normas/{id}` or `/jurisprudencia/{id}` based on node type
    - Zoom controls (AppKit 4 button group)
    - "Demasiadas relaciones — zoom en la norma" empty state when `truncated: true`

- [ ] **Create** `ruling-quick-list.component.ts`
    - Reuses `ruling-result-card` (from F1.3); shows first 5 related rulings
    - "Ver todas" link → `/jurisprudencia?norm={id}` (pre-filters search by norm)

### Routing

- [ ] Route `/jurisprudencia/:id` → `ruling-detail-shell` (reuses shell; parametrized by type)
