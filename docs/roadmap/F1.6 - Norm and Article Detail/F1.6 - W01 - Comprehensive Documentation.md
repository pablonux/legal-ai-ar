## 1. Feature Overview

Detailed view of a **legal norm** (or a tax document from F1.5), including:

- **Metadata panel** — norm number, name, type, dates, issuing body, hierarchy level, in-force status.
- **Article browser** — paginated/scrollable list of articles with article-level search.
- **Relationship graph** — interactive Cytoscape.js graph showing norms/rulings linked via SQL Graph
  edges (`Amends`, `Repeals`, `Regulates`, `Interprets`, `CitesCaseLaw`, `References`).

MVP coverage: **~85%** — the detail view exists but the relationship graph and article-level search
need to be wired to the GraphRAG traversal service and aligned to Contracts/Minimal API.

## 3. Backend Architecture

### 3.1 Endpoints

| Method | Route                                   | Auth     | Description                                   |
| ------ | --------------------------------------- | -------- | --------------------------------------------- |
| GET    | `/api/statutes/{id}`                    | PwCStaff | Norm metadata + article count                 |
| GET    | `/api/statutes/{id}/articles`           | PwCStaff | Paginated list of articles                    |
| GET    | `/api/statutes/{id}/articles/{artId}`   | PwCStaff | Single article full text                      |
| GET    | `/api/statutes/{id}/relations`          | PwCStaff | Graph traversal: connected norms + rulings    |
| GET    | `/api/statutes/{id}/case-law`           | PwCStaff | Rulings that interpret/apply this norm (hybrid)|
| GET    | `/api/rulings/{id}`                     | PwCStaff | Ruling metadata (existing, align to Contracts)|
| GET    | `/api/rulings/{id}/relations`           | PwCStaff | Graph: norms this ruling applies + cites      |

The `/relations` endpoint performs a **SQL Graph traversal** (1–2 hops) and returns a `GraphResponse`
suitable for Cytoscape.js rendering.

### 3.2 Contracts

```csharp
// Contracts/Statutes/StatuteDetailResponse.cs
public sealed record StatuteDetailResponse(
    int      Id,
    string   NormNumber,
    string   Name,
    string   NormType,
    int      HierarchyLevel,
    string   LawBranch,
    string   TerritorialScope,
    string   IssuingBody,
    DateOnly? EnactmentDate,
    DateOnly? PublicationDate,
    bool     IsInForce,
    string?  OriginalTextUrl,
    string?  Summary,
    int      ArticleCount);

// Contracts/Graph/GraphResponse.cs
public sealed record GraphResponse(
    IReadOnlyList<GraphNode> Nodes,
    IReadOnlyList<GraphEdge> Edges);

public sealed record GraphNode(
    string Id,
    string Label,
    string Type,    // "norm"|"ruling"|"article"
    string? Url);

public sealed record GraphEdge(
    string Id,
    string Source,
    string Target,
    string RelationType);  // Amends|Repeals|Interprets|Applies|CitesCaseLaw|etc.
```

### 3.3 Graph traversal (Infrastructure)

```
LegalAiAr.Infrastructure/
└── Graph/
    └── LegalGraphService.cs    # Executes MATCH queries on SQL Graph, maps to GraphResponse
```

Maximum depth: 2 hops. Maximum nodes returned: 50 (configurable). Nodes > 50 trigger a "too many
relations — zoom in" message.

```
├── components/
│   ├── norm-detail-shell/         # Tab container + metadata header

│   └── norm-detail.service.ts
├── models/
│   └── norm-detail.model.ts
├── norm-detail.routes.ts          # /normas/:id
└── index.ts
```

Also: `ruling-detail` route at `/jurisprudencia/:id` (shares most components via parametrized approach).

### 4.2 Cytoscape.js integration

```typescript
// Cytoscape config — already in MVP, align to new GraphResponse DTO

```

Clicking a node navigates to the corresponding detail page (deep link).
