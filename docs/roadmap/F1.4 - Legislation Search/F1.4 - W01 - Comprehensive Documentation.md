## 1. Feature Overview

Production-ready legislation search for Argentine norms (laws, decrees, resolutions, etc.) backed by
the hybrid search over `idx-legal-norms` (BM25 + 3072-d vectors). MVP coverage is **~90%** — this
feature aligns the backend to the F0.0 foundation (Contracts DTOs, Minimal API endpoints, platform
auth) and delivers the polished Angular 19 search UI using AppKit 4 components.

---

## 2. Search Capabilities

| Capability               | Implementation                                      |
| ------------------------ | --------------------------------------------------- |
| Full-text + semantic     | Hybrid search on `idx-legal-norms` (BM25 + vectors) |
| Kelsen pyramid filter    | `hierarchyLevel` facet (1 = Constitution … 5 = Res) |
| Law branch filter        | `lawBranch` facet (tributario, civil, laboral, …)   |
| Territorial scope filter | `territorialScope` (nacional, provincial, caba)     |
| In-force toggle          | `isInForce` filter                                  |
| Date range               | `publicationDate` range filter                      |
| Issuing body             | `issuingBody` facet + filter                        |
| Sort                     | Relevance (default) / date desc / date asc          |
| Pagination               | Cursor-based, 20 per page                           |

## 3. Backend Architecture

### 3.1 Endpoints (already scaffolded in F0.0-W09)

| Method | Route                          | Auth     | Description                            |
| ------ | ------------------------------ | -------- | -------------------------------------- |
| POST   | `/api/statutes/search`         | PwCStaff | Hybrid search over `idx-legal-norms`   |
| GET    | `/api/statutes/{id}`           | PwCStaff | Norm detail (→ F1.6 extends this)      |
| GET    | `/api/statutes/facets`         | PwCStaff | Returns available facet counts         |
| GET    | `/api/statutes/pyramid`        | PwCStaff | Kelsen pyramid summary                 |

All endpoints already exist as Minimal API stubs from F0.0-W09. This feature wires them to the
real search service and migrates their request/response types to `LegalAiAr.Contracts`.

### 3.2 Contracts

```csharp
// Contracts/Statutes/SearchStatutesRequest.cs
public sealed record SearchStatutesRequest(
    string?  Query,
    string?  LawBranch,
    string?  TerritorialScope,
    string?  IssuingBody,
    int?     HierarchyLevel,
    bool?    IsInForce,
    DateOnly? PublishedFrom,

    string   SortBy = "relevance",   // relevance|date_desc|date_asc
    int      Page = 1,
    int      PageSize = 20);

// Contracts/Statutes/StatuteSummary.cs
public sealed record StatuteSummary(
    int      Id,
    string   NormNumber,
    string   Name,
    string   NormType,
    int      HierarchyLevel,
    string   LawBranch,
    string   TerritorialScope,
    DateOnly? PublicationDate,
    bool     IsInForce,

// Contracts/Statutes/SearchStatutesResponse.cs  
public sealed record SearchStatutesResponse(
    IReadOnlyList<StatuteSummary> Results,
    int TotalCount,
    IReadOnlyDictionary<string, IReadOnlyList<FacetValue>> Facets,
    string? NextCursor);
```

## 4. Frontend Architecture

```
├── models/
│   └── legislation.model.ts
├── legislation-search.routes.ts      # lazy-loaded at /normas
└── index.ts
```

Route: `/normas`

### 4.2 State

```typescript
@Injectable({ providedIn: 'root' })
export class LegislationSearchService {
  readonly filters  = signal<SearchStatutesRequest>(defaultFilters);
  readonly results  = signal<SearchStatutesResponse | null>(null);
  readonly loading  = signal(false);

  search(overrides?: Partial<SearchStatutesRequest>): void;
  loadMore(): void;    // cursor pagination
  clearFilters(): void;
}
```

### 4.3 UI notes

- Facet sidebar uses AppKit 4 checkbox groups and accordion panels.
- Results use AppKit 4 card components with highlighted text snippets.
- Pyramid view is an optional SVG visual — filterable by clicking a hierarchy level.
- Search input uses debounce (300 ms) and fires on Enter or button click.
