# F1.4 - W03 - Frontend Legislation Search Feature

> **Feature:** F1.4 - Legislation Search
> **Release:** 1.0 | **Sprint:** S01
> **Type:** frontend | **Priority:** High
> **Estimate:** 5 story points
> **Assignable to:** Frontend Dev

---

## Description

Build the Angular 19 legislation search feature at `/normas`. Search bar + facet sidebar + result
cards + Kelsen pyramid visual. AppKit 4 components, Signals state management.

---

## Tasks

### Feature scaffold

- [ ] Generate `frontend/projects/features/legislation-search/` via `schema-la:feature`

### Models

- [ ] **Create** `legislation.model.ts`
    ```typescript
    export interface SearchStatutesRequest {
      query?: string; lawBranch?: string; territorialScope?: string;
      issuingBody?: string; hierarchyLevel?: number; isInForce?: boolean;
      publishedFrom?: string; publishedTo?: string; sortBy: string; page: number; pageSize: number;
    }
    export interface StatuteSummary {
      id: number; normNumber: string; name: string; normType: string;
      hierarchyLevel: number; lawBranch: string; territorialScope: string;
      publicationDate?: string; isInForce: boolean; issuingBody: string; highlightedSummary: string;
    }
    ```

### Service

- [ ] **Create** `legislation-search.service.ts`
    - Signals: `filters`, `results`, `loading`
    - Methods: `search(overrides?)`, `loadMore()`, `clearFilters()`
    - `getPyramidData(): Observable<PyramidLevel[]>` — calls `GET /api/statutes/pyramid`

### Components

- [ ] **Create** `legislation-search-shell.component.ts` — two-panel layout; title "Buscar Normas"

- [ ] **Create** `legislation-filters.component.ts`
    - AppKit 4 accordion panels:
      - Rama del derecho: checkbox group from `facets.lawBranch`
      - Ámbito territorial: radio buttons (Nacional / Provincial / CABA)
      - En vigor: toggle switch
      - Nivel Kelsen: slider 1–5 or select
      - Fecha de publicación: date range
    - "Limpiar filtros" button

- [ ] **Create** `pyramid-explorer.component.ts`
    - SVG pyramid with 5 levels; each level shows label + count
    - Click on a level → sets `hierarchyLevel` filter
    - Levels: Constitución (1), Tratados (2), Leyes (3), Decretos (4), Resoluciones (5)

- [ ] **Create** `statute-result-card.component.ts`
    - Norm number badge, name, type chip, law branch chip, in-force badge (green/red)
    - Highlighted summary (renders `<mark>` safely via `innerHTML` binding on trusted content)
    - Click → navigate to `/normas/{id}`

- [ ] **Create** `legislation-results.component.ts`
    - List of `statute-result-card` with total count header
    - Cursor pagination ("Cargar más") or paged navigation
    - Empty state: "No se encontraron normas"

### Routing

- [ ] Route `/normas` → `legislation-search-shell` (protected by `authGuard`)
- [ ] Route `/normas/:id` → (navigates to norm-detail feature — F1.6)

### Tests

- [ ] Unit: `legislation-search.service.ts` — search calls POST with correct body
- [ ] Unit: `legislation-filters.component.ts` — filter changes emit to service
- [ ] Unit: `pyramid-explorer.component.ts` — renders 5 levels; click sets filter
- [ ] E2E smoke: navigate to `/normas`, type query, see results

### Verification

- [ ] `ng build` zero errors; `npm run lint` zero warnings
- [ ] Filters sidebar updates results reactively
- [ ] Pagination loads next page without losing active filters
- [ ] DoD: STATUS.md updated
